# Fichiers protégés

## Ce que c'est

Les fichiers que l'étudiant ne doit pas toucher : les critères de correction,
le workflow de correction, l'énoncé. Les suspects habituels (`criteria.yml`,
`README.md`, `.github/workflows/grading.yml`) sont pré-cochés automatiquement
quand ils existent dans le dépôt source.

## Ce que fait la protection

Si un étudiant committe une modification d'un fichier protégé, la plateforme
la rétablit automatiquement en quelques secondes (le contenu restauré vient du
snapshot du devoir). Les altérations répétées sont limitées et signalées.

## Ajuster la liste

Dépliez la section et cochez/décochez les fichiers dans l'arborescence. La
liste reste modifiable après création, depuis le dialogue d'édition du devoir.
